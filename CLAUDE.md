# Forge — Personal Workout App

## Quick start

```bash
cp .env.example .env          # fill in DATABASE_URL and ANTHROPIC_API_KEY
npx prisma migrate dev --name init
npm run dev                   # http://localhost:3000
```

## Required env vars

| Var | Description |
|-----|-------------|
| `DATABASE_URL` | Postgres connection string (Neon, Supabase, Railway, or local) |
| `ANTHROPIC_API_KEY` | From https://console.anthropic.com/settings/keys |

## Prisma schema (prisma/schema.prisma)

### Profile (singleton, id=1)
Stores user preferences. Created once via `/onboarding`.
- `goal`: STRENGTH | MUSCLE | FAT_LOSS | ENDURANCE | GENERAL | ATHLETIC
- `experience`: NEW | INTERMEDIATE | ADVANCED
- `trainingLocation`: FULL_GYM | HOME_DUMBBELLS | MINIMAL
- `daysPerWeek`: training days per week
- `units`: KG | LB

### CheckIn (one per day)
Captured before generating each day's workout.
- `energy` 1–5, `hydration` 1–4
- `painAreas`: string array of named body regions
- `fuelLevel`: UNDERFUELED | LIGHT | FED | HEAVY
- `minutesAvailable`: session time budget

### Workout
Generated by the AI for each CheckIn. JSON fields:
- `warmup`: string[]
- `blocks`: `{ name: string; exercises: Exercise[] }[]`
- `cooldown`: string[]
- `status`: PLANNED → COMPLETED or SKIPPED

Exercise shape: `{ name, scheme, rpe, rest, note, swap, muscleGroup }`

### ExerciseLog
Normalised rows written on workout completion, one per exercise.
Used for progressive overload queries (`lastSimilarSession` lookup).

## Generator contract (lib/generator.ts)

```ts
generateWorkout(profile, recentWorkouts, checkIn)
  → { output: WorkoutOutput, rawOutput: string }
```

- Fetches last 7 workouts with exerciseLogs and checkIn.
- Calls `claude-sonnet-4-6` via tool use with `log_workout` tool.
- Validates response against `WorkoutOutputSchema` (Zod).
- Retries once on validation failure.
- Throws with a user-facing message on second failure.

## Server actions (app/actions.ts)

| Action | Description |
|--------|-------------|
| `saveProfile(formData)` | Upserts Profile row, redirects to /today |
| `submitCheckIn(prev, formData)` | Creates CheckIn + calls generator, returns workoutId |
| `completeWorkout(formData)` | Sets status=COMPLETED, writes ExerciseLogs, records difficulty |
| `skipWorkout(workoutId)` | Sets status=SKIPPED |

## Screens

| Route | Description |
|-------|-------------|
| `/` | Redirects to /onboarding (no profile) or /today |
| `/onboarding` | One-time setup: goal, experience, location, days, notes |
| `/today` | Check-in chips → generate → display plan → mark complete |
| `/history` | Last 30 sessions + 14-session readiness trend SVG |

## Seeding your profile (first run)

After migration, visit http://localhost:3000 — the onboarding form runs automatically if no Profile exists.

Or seed via Prisma Studio:
```bash
npx prisma studio
```
